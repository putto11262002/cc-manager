# OpenAPI Docs (v0)

## Summary

Adds a generated OpenAPI 3.1 spec and a modern docs UI without Swagger UI.

## User-Facing Endpoints

- `GET /openapi.json` — OpenAPI 3.1 spec generated from the Hono app.
- `GET /docs` — Scalar API Reference UI rendering the spec.

## Dependencies

- `hono-openapi` — generates OpenAPI specs from the Hono app and validation schemas.
- `@hono/standard-validator` — required peer for `hono-openapi`.
- `@scalar/hono-api-reference` — modern docs UI for OpenAPI specs.

## Implementation Notes

- The spec endpoint is registered in `src/index.ts` using `openAPIRouteHandler(app, ...)`.
- `includeEmptyPaths: true` is enabled so routes show up even before schemas are fully annotated.
- The docs endpoint uses Scalar and points at the local spec URL (`/openapi.json`).
- Existing routes still use `@hono/zod-validator` for request validation; no refactor required.
- Schema enrichment can be added later per-route via `hono-openapi` helpers (see Future Work).

## Future Work / Extensions

- **Per-route schemas**: Swap validators to `hono-openapi`'s `validator(...)` and add
  `describeRoute(...)` to document request/response shapes.
- **Tags & auth**: Add tags, servers, security schemes, and response models in the
  `documentation` config.
- **Versioned specs**: Add `/openapi.v1.json` or `/openapi.yaml` if needed.
- **Custom Scalar theming**: Provide a theme config or a custom HTML shell.
